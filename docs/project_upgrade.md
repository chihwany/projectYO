# ProjectYO 업그레이드 기능 기획서

## 문서 개요

이 문서는 ProjectYO의 핵심 서비스(중고나라·번개장터 통합 검색 및 키워드 알림, 당근 지역 검색)를 기반으로, 더 많은 사용자가 지속적으로 사용하는 앱이 되기 위한 추가 기능 아이디어를 정리한 기획서입니다.

---

## 1. 핵심 가치 재정의

> **"중고 거래에 들이는 시간과 노력을 최소화한다"**

현재 사용자는 원하는 매물을 찾기 위해 매일 여러 플랫폼을 직접 돌아다니며 수동으로 검색해야 합니다. ProjectYO는 이 반복적인 수고를 대신해 줌으로써 사용자의 시간과 에너지를 절약합니다.

### 앱의 세 가지 핵심 기능

| 핵심 기능 | 설명 |
|---|---|
| **통합 검색** | 중고나라·번개장터를 한 번에 검색하여 매물 비교 |
| **키워드 알림** | 매분 최신 매물 수집 → 키워드 매칭 → FCM 푸시 알림 |
| **당근 지역 검색** | 전국 어느 지역이든 선택하여 당근 매물 검색 |

#### 2-1. 제외 키워드 설정

**배경**
"맥북" 키워드를 등록하면 맥북에어, 맥북 케이스, 맥북 거치대 등 원하지 않는 매물이 대량으로 알림으로 들어옵니다. 알림 피로도(notification fatigue)가 생기면 사용자는 알림을 꺼버리거나 앱을 삭제합니다.

**기능 설명**
키워드 등록/수정 화면에서 "제외할 단어" 목록을 추가로 입력할 수 있습니다. 상품 제목에 제외 단어가 포함되어 있으면 알림에서 자동으로 필터링합니다.

**예시**
- 키워드: `맥북`
- 제외 키워드: `에어`, `케이스`, `거치대`, `파우치`
- 효과: "맥북 프로 14인치 M3" 같은 매물만 알림

**데이터 모델 변경**
```sql
-- keywords 테이블에 컬럼 추가
ALTER TABLE keywords ADD COLUMN exclude_keywords TEXT[] DEFAULT '{}';
```

#### 2-2. 관심 매물 저장 & 상태 추적

**배경**
"나중에 살까 생각 중인" 매물을 별도로 저장해 두고 싶은 니즈가 강합니다. 특히 "내가 찜했는데 예약이 됐다" → "빠르게 대안을 찾아야 한다"는 상황에서 앱의 가치가 극대화됩니다.

**기능 설명**
- 검색 결과 또는 알림 이력에서 매물 카드의 ❤️ 버튼으로 저장
- 저장된 매물의 상태 변화(판매중 → 예약중 → 판매완료)를 주기적으로 체크
- 상태 변화 시 즉시 푸시 알림 발송

**상태 변화 알림 메시지 예시**
```
❤️ 찜한 매물이 예약 상태로 변경됐어요
"아이폰 15 Pro 256GB (1,050,000원)"
비슷한 매물 보러 가기 →
```

**데이터 모델**
```sql
CREATE TABLE saved_products (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  product_id  VARCHAR(100) NOT NULL,
  platform    VARCHAR(20) NOT NULL,       -- 'bunjang' | 'joongna' | 'daangn'
  title       TEXT NOT NULL,
  price       INTEGER,
  url         TEXT NOT NULL,
  image_url   TEXT,
  last_status VARCHAR(20) DEFAULT '판매중',  -- 마지막 확인된 상태
  saved_at    TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, platform, product_id)
);
```

**스케줄러 추가 작업**
- 저장된 매물 목록을 10분마다 상태 재확인
- 이전 상태와 달라졌으면 FCM 발송

#### 2-3. 당근 즐겨찾기 지역 저장

**배경**
당근 탭에서 매번 지역을 새로 검색하고 선택하는 과정이 번거롭습니다. 특히 "서울 강남구 일대 전체"처럼 여러 동을 한꺼번에 보고 싶은 경우 매번 재설정해야 하는 불편함이 큽니다.

**기능 설명**
- 지역 선택 후 "이 지역 즐겨찾기 저장" 버튼 제공
- 저장된 즐겨찾기 지역 목록에서 탭 한 번으로 바로 검색 실행
- 즐겨찾기는 비로그인 사용자도 로컬 저장 가능

**즐겨찾기 저장 데이터 구조 (Flutter 로컬)**
```json
[
  {
    "label": "고양시 덕양구 능곡동",
    "location_ids": [1540],
    "depth": 3
  },
  {
    "label": "서울 강남구 전체",
    "location_ids": [1001, 1002, 1003, ...],
    "depth": 2
  }
]
```

#### 2-4. 검색 히스토리 & 빠른 재검색

**배경**
중고 거래 특성상 같은 키워드를 반복 검색하는 패턴이 매우 강합니다. 매번 타이핑하는 불편함을 제거하면 앱 사용 속도와 만족도가 높아집니다.

**기능 설명**
- 검색창 탭 시 최근 검색어 최대 10개 표시
- 검색어 탭 → 즉시 재검색
- 검색어 개별 삭제 및 전체 삭제 지원
- 비로그인 사용자도 로컬 저장

#### 2-5. 가격 히스토리 & 시세 분석 ⭐ 최고 차별화 포인트

**배경**
사용자들이 중고 거래에서 가장 고민하는 것은 "이 가격이 적당한가?"입니다. 다른 중고 거래 플랫폼(번개장터, 중고나라, 당근 앱 자체)은 이 기능을 제대로 제공하지 않습니다. 이 기능이 이 앱만의 강력한 차별화 요소가 됩니다.

**기능 설명**
- 수집된 매물 데이터를 누적 저장하여 키워드별 가격 데이터 축적
- 검색 결과 상단에 "이 키워드의 시세 요약" 카드 표시
- 개별 매물 카드에 시세 대비 가격 배지 표시

**시세 요약 카드 표시 예시**
```
📊 "아이폰 15 Pro 256GB" 시세
최근 30일 평균: 1,230,000원
최저가: 980,000원 / 최고가: 1,500,000원
━━━━━━━━━━━━━━━━━━━━━
현재 검색 결과 중 평균보다 저렴한 매물: 3건
```

**개별 매물 배지 예시**
```
[시세보다 15% 저렴] ← 초록색 배지
[시세보다 12% 비쌈] ← 빨간색 배지
```

**데이터 모델**
```sql
CREATE TABLE price_history (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  keyword     VARCHAR(100) NOT NULL,
  platform    VARCHAR(20) NOT NULL,
  product_id  VARCHAR(100) NOT NULL,
  price       INTEGER NOT NULL,
  collected_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(platform, product_id)  -- 동일 상품 중복 저장 방지
);

CREATE INDEX idx_price_history_keyword ON price_history(keyword, collected_at DESC);
```

**시세 계산 API**
```
GET /search/stats?keyword=아이폰+15+Pro
Response:
{
  "keyword": "아이폰 15 Pro",
  "period_days": 30,
  "count": 847,
  "avg_price": 1230000,
  "median_price": 1200000,
  "min_price": 980000,
  "max_price": 1500000,
  "price_trend": "하락중"  // 상승중 | 하락중 | 안정
}
```

#### 2-6. 가격 Drop 알림

**배경**
"지금 가격은 비싸지만 조금만 내리면 살 텐데"라는 상황이 매우 자주 발생합니다. 관심 매물을 찜해 두고 판매자가 가격을 낮추면 즉시 알려주는 기능으로, 구매 전환율을 높입니다.

**기능 설명**
- 매물 저장 시 "이 가격 이하로 내리면 알려줘" 목표 가격 입력 옵션
- 상태 추적(2-2) 스케줄러에서 가격 변동도 함께 체크
- 목표 가격 이하로 내려가면 즉시 FCM 발송

**알림 메시지 예시**
```
💰 찜한 매물 가격이 내려갔어요!
"아이폰 15 Pro 256GB"
1,200,000원 → 1,050,000원 (▼150,000원)
지금 바로 확인하기 →
```

**데이터 모델 변경**
```sql
-- saved_products 테이블에 컬럼 추가
ALTER TABLE saved_products ADD COLUMN target_price INTEGER; -- NULL이면 가격 알림 없음
ALTER TABLE saved_products ADD COLUMN last_price INTEGER;   -- 마지막 확인된 가격
```

#### 2-7. 키워드 통계 대시보드

**배경**
내가 등록한 키워드에 대해 "언제 매물이 많이 올라오나", "가격이 오르고 있나 내리고 있나"를 알면 전략적으로 앱을 활용할 수 있습니다. 이 기능은 앱을 단순 알림 도구가 아닌 "중고 거래 분석 도구"로 포지셔닝하게 해줍니다.

**기능 설명**
- 키워드별 대시보드 화면 제공
- 표시 항목:
  - 오늘 / 이번 주 / 이번 달 매물 발생 건수
  - 평균 가격 추이 선 그래프 (최근 30일)
  - 요일별 / 시간대별 매물 발생 히트맵
  - 가장 매물이 많이 올라오는 시간대 강조 표시

**히트맵 예시 (시간대별 매물 발생)**

#### 2-8. 매물 공유 기능

**배경**
"이거 어때요?"처럼 주변 사람과 매물을 공유하는 행동은 중고 거래에서 매우 자주 일어납니다. 공유 기능이 있으면 앱 내에서 자연스러운 바이럴이 발생하고, 공유받은 사람이 앱을 설치하는 획득 채널이 됩니다.

**기능 설명**
- 매물 카드 길게 누르기 또는 공유 버튼 탭
- 공유 방식: 카카오톡, 문자, 링크 복사, 기타(OS 공유 시트)
- 공유 내용: 매물 제목, 가격, 이미지, 링크

**공유 텍스트 예시**
```
[번개장터] 아이폰 15 Pro 256GB - 1,050,000원
https://m.bunjang.co.kr/products/123456789

ProjectYO 앱에서 찾았어요 →
```

#### 2-9. 플랫폼 간 동일 상품 가격 비교 뷰

**배경**
같은 상품이 번개장터에는 100만원, 중고나라에는 95만원에 올라와 있을 수 있습니다. 이 차이를 앱 안에서 한눈에 보여주는 것 자체가 이 앱을 쓰는 가장 강력한 이유가 됩니다.

**기능 설명**
- 검색 결과 화면 상단에 "플랫폼별 최저가 요약" 카드 표시
- 두 플랫폼 가격 차이와 절약 가능 금액 강조

**비교 카드 예시**
```
💡 플랫폼별 최저가 비교 — "아이폰 15 Pro"
번개장터  최저 980,000원 (32건)
중고나라  최저 950,000원 (18건) ← 여기가 더 저렴
━━━━━━━━━━━━━━━━━━━━
최대 30,000원 절약 가능
```

#### 2-12. 판매자 신뢰도 표시

**배경**
중고 거래에서 판매자 신뢰도는 구매 결정의 핵심 요소입니다. 특히 처음 보는 판매자의 경우 플랫폼 앱을 직접 열어야만 확인할 수 있는 불편함이 있습니다.

**기능 설명**
- 번개장터: 판매자 평점, 판매 횟수 매물 카드에 표시 (API 응답에 포함된 경우)
- 중고나라: 인증 배지 여부 표시 (본인인증, 계좌인증 등)
- 당근: 매너 온도 표시 (수집 가능한 경우)

#### 2-13. 신규 가입자 온보딩 키워드 추천

**배경**
가입 직후 키워드를 등록하지 않으면 앱의 핵심 가치(알림)를 경험하지 못하고 이탈할 가능성이 높습니다. 첫 번째 알림을 빠르게 경험시키는 것이 앱 정착률에 결정적입니다.

**기능 설명**
- 회원가입 완료 직후 온보딩 화면 표시
- "지금 많이 검색되는 키워드 TOP 10" 추천 (서버에서 집계)
- 이 시즌에 핫한 카테고리 추천 (예: 봄철 → 자전거, 아이패드)
- 추천 키워드 탭 → 즉시 등록 → 첫 알림 대기 상태 안내

**온보딩 플로우**
```
회원가입 완료
      ↓
"어떤 매물을 찾고 계세요?" 화면
      ↓
인기 키워드 칩 목록 표시 (탭으로 선택)
      ↓
직접 입력도 가능
      ↓
선택한 키워드 일괄 등록
      ↓
"등록 완료! 새 매물이 올라오면 바로 알려드릴게요 🔔" 안내
```

#### 2-14. 관심 카테고리 기반 피드

**배경**
키워드 알림 방식은 "이미 원하는 것을 아는" 사용자에게 최적화되어 있습니다. 하지만 "전자기기 매물을 둘러보고 싶다"처럼 카테고리 단위로 탐색하고 싶은 니즈도 있습니다.

**기능 설명**
- 관심 카테고리 설정 (번개장터·중고나라 카테고리 목록 기반)
- 메인 피드에서 설정한 카테고리의 최신 매물을 타임라인으로 표시
- 당기기(pull-to-refresh)로 최신 매물 갱신
- 이미 본 매물은 회색으로 표시

## 4. 빠른 성장을 위한 핵심 전략

### 첫 번째 알림 경험을 최대한 빠르게

사용자 이탈의 가장 큰 원인은 **"앱의 가치를 경험하기 전에 떠나는 것"**입니다. 가입 → 키워드 등록 → 첫 알림 수신까지의 과정을 최대한 짧고 마찰 없이 만드는 것이 최우선입니다.

- 비로그인 상태에서도 검색은 가능하게 유지 (진입 장벽 낮춤)
- 온보딩에서 인기 키워드 추천으로 키워드 등록 유도
- 첫 알림 수신 후 "이런 매물이 올라왔어요" 친절한 안내

### 가격 시세 기능이 바이럴의 핵심

"이 앱 쓰면 중고 시세를 알 수 있다"는 한 문장 설명이 가능해지면 자연스러운 입소문이 발생합니다. 가격 데이터 수집은 MVP 시점부터 시작해야 v1.1에서 충분한 데이터로 기능을 제공할 수 있습니다.

> **중요**: 가격 히스토리 데이터 수집은 MVP 출시 시점부터 백그라운드에서 시작할 것. 별도 화면 노출은 v1.1에서 해도 되지만, 데이터 축적은 즉시 시작해야 합니다.

### 공유 기능으로 자연 획득

별도 마케팅 비용 없이 "이 매물 어때요?" 공유 하나가 신규 사용자를 데려오는 채널이 됩니다. 공유 링크에 앱 다운로드 배너를 붙이면 효과를 극대화할 수 있습니다.

## 5. 데이터 모델 추가 변경 요약

기존 스키마에 추가/변경이 필요한 항목들입니다.

```sql
-- 1. keywords 테이블 — 제외 키워드 추가
ALTER TABLE keywords
  ADD COLUMN exclude_keywords TEXT[] DEFAULT '{}';

-- 2. 관심 매물 저장 테이블 (신규)
CREATE TABLE saved_products (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id      UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  product_id   VARCHAR(100) NOT NULL,
  platform     VARCHAR(20) NOT NULL,
  title        TEXT NOT NULL,
  price        INTEGER,
  target_price INTEGER,         -- 가격 Drop 알림용 목표 가격
  last_price   INTEGER,         -- 마지막 확인 가격
  url          TEXT NOT NULL,
  image_url    TEXT,
  last_status  VARCHAR(20) DEFAULT '판매중',
  saved_at     TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, platform, product_id)
);

-- 3. 가격 히스토리 테이블 (신규)
CREATE TABLE price_history (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  keyword      VARCHAR(100) NOT NULL,
  platform     VARCHAR(20) NOT NULL,
  product_id   VARCHAR(100) NOT NULL,
  price        INTEGER NOT NULL,
  collected_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(platform, product_id)
);

CREATE INDEX idx_price_history_keyword ON price_history(keyword, collected_at DESC);
```

## 6. 결론

ProjectYO의 가장 강력한 차별화 포인트는 **멀티 플랫폼 통합**입니다. 이 강점을 극대화하는 방향으로 기능을 추가해야 합니다.

우선순위를 한 문장으로 정리하면:

> **MVP에서 마찰을 줄이고(제외 키워드, 즐겨찾기), v1.1에서 시세 분석으로 차별화하고(가격 히스토리, Drop 알림), v1.2에서 바이럴을 만든다(공유, 가격 비교).**

각 단계를 순서대로 검증하면서 개발하면 무리 없이 성장 가능한 서비스를 만들 수 있습니다.
